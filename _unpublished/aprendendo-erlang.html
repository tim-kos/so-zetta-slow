Erlang é uma linguagem de programação funcional. Da wikipedia:

In computer science, functional programming is a programming paradigm that treats computation as the evaluation of mathematical functions and avoids state and mutable data. It emphasizes the application of functions, in contrast to the imperative programming style, which emphasizes changes in state.

Basicamente, as funções nesse tipo de programação não tem estado e podem ser usadas aproximadamente como se estivesse mexendo com funções no sentido matemático. Ainda estou estudando e aprendendo como isso é usado, mas isso quer dizer (corrijam-me se estiver errado) que funções são objetos de primeira ordem na linguagem, como em JavaScript (só lembrar dos lambdas que o Douglas Crackford cita em The Good Parts). Isso permite coisas como map e reduce (ou Enumerable#Map e Enumerable#Inject do Ruby) e uso intensivo de recursão.

Além disso, Erlang se usa de Pattern Matching para mexer com igualdades e atribuições. Ainda não sei direito as implicações disso, mas escreverei a respeito assim que for possível.
